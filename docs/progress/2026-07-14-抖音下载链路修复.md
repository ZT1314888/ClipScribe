# 抖音下载链路修复（a_bogus 签名）

> 日期：2026-07-14
> 分支：`fix/douyin-download-abogus`
> 提交：`9120c81`
> 关联：ADR-0002（已修订）、[docs/douyin-download.md](../douyin-download.md)

## 现象

粘贴抖音链接下载失败，任务转 `failed`，错误信息为：

```
抖音解析失败：RetryError[<Future ... raised ValueError>]
```

`jingxuan?modal_id=...` 精选页链接和普通分享链接都失败。

## 根因

旧依赖 `douyin-tiktok-scraper`（Evil0ctal，v1.2.9）已失效：

1. **签名过时（主因）**：它仍用旧的 `X-Bogus` 签名，而抖音 detail 接口
   （`/aweme/v1/web/aweme/detail/`）已要求 `a_bogus` 签名。服务端返回的数据里
   没有 `aweme_detail`，库内 `response['aweme_detail']` 抛 `KeyError` →
   被包装成 `ValueError` → `@retry` 重试 4 次后抛 `RetryError`。
2. **链接格式不认**：库只处理 `discover?modal_id=`，不认 `jingxuan?modal_id=`。
   （即便修了这条，第 1 条也照样失败。）

与 Cookie 无关——排查确认 `.env` 里的 Cookie 完整（含 `ttwid`/`sessionid`/`sid_guard`
等 httpOnly 项）。

## 排查过程（关键结论）

对比 seekin.ai 等下载站「粘链接就能下」的原理，抖音有两条取数路径：

- **私有 API**（detail 接口）：要 `a_bogus` 签名，抖音持续更新，开源库跟不上就死。
- **分享页 HTML**：实测已返回 `_$jsvmprt` JS 反爬挑战页，数据不再内联，curl 拿不到。

结论：绕不开签名。下载站靠的是「自己养一套持续维护的 a_bogus」或「真实浏览器内核跑过反爬」。
对内部 MVP，最务实的是站在活跃维护的开源实现肩膀上。

评估过的候选：

- **yt-dlp**：认抖音链接，但 `--cookies-from-browser edge` 在本机撞 DPAPI 加密
  （App-Bound Encryption），需插件导 `cookies.txt` 才行。
- **douyin-downloader（jiji262，MIT，2026-07 仍活跃）**：自带一份纯 Python 的 `a_bogus`
  实现（`utils/abogus.py`）。单条视频走纯 HTTP + 签名，**不碰 Playwright**
  （其浏览器兜底仅用于「主页批量翻页被风控」，与单视频无关）。实测用 `.env` 的 Cookie
  取到无水印地址并下载 205MB 真实 mp4，全程不开浏览器。→ **选它。**

## 方案

vendored douyin-downloader 的 API 层子集，加自研适配层。

- **为什么 vendored 而非 pip 安装**：上游是应用不是库，pip 会拖进 imageio-ffmpeg(29MB)/
  rich/aiosqlite 等无关重依赖并触发重 import 链。只抽单视频解析真正需要的 8 个文件到
  `app/services/douyin_vendor/`，第三方依赖收敛为 `aiohttp` + `gmssl`。
- **不撞 MVP 边界**：单视频纯 HTTP，无浏览器、无 Redis/Celery/Postgres/S3，故不需新 ADR，
  只修订 ADR-0002。
- **Cookie 策略不变**：仍从 `settings.douyin_cookie` 环境注入，`parse_cookie_header`
  转 dict，不落盘不入库。
- **接口不变**：`download(url, task_id) -> DownloadResult` 与错误映射（`DownloadError`）
  保持原样，pipeline / API / 状态机零改动。

### 链路

```
短链跟随重定向 → URLParser 取 aweme_id → DouyinAPIClient.get_video_detail
→ 从 aweme_detail 抠无水印地址（play_addr 优先，回退 h264/download_addr/bit_rate）
→ 流式落盘 data/media/<task_id>/video.mp4
```

图集/私密/风控/主页合集等能力边界外输入，一律映射 `DownloadError`，提示改用本地上传兜底。

## 验证

- 全套 22 个测试通过（`uv run pytest`）。
- 真实端到端：用 `.env` Cookie 走项目正式代码路径解析 + 下载你截图那条链接，
  成功落盘 205MB 真实 mp4（标题「地推遇良师，茶遇知心人…」、作者「陌的猫」）。

## 后续注意

- **Cookie 会过期**：解析持续失败时先到 `.env` 重贴新 Cookie。
- **跟随上游**：抖音改签名时同步 `douyin_vendor/`，方法见其 `UPSTREAM.md`。
