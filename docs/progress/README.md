# 进度文档（docs/progress/）

本目录记录项目的**进度快照**与**重要变更的实施记录**，回答「现在做到哪了、某次改动为什么这么做」。

与其它文档的分工：

| 文档 | 回答的问题 | 位置 |
|---|---|---|
| 方案 / 补充计划 | 要做成什么样（目标、边界） | `docs/短视频文案系统_MVP*.md` |
| ADR | 某个技术决策为什么这么定 | `docs/adr/` |
| **进度文档（本目录）** | **现在做到哪了、某次改动怎么落地的** | `docs/progress/` |
| 领域术语 | 名词怎么定义 | `CONTEXT.md` |

## 约定

- 进度总览随里程碑更新，保持「当前状态」这一份最新即可。
- 重要变更记录按 `YYYY-MM-DD-简述.md` 命名，一次成文后一般不再改（历史留痕）。
